# CPE Manager v0.3.1

This alpha release fixes the Huawei login flow seen in the new HAR and upgrades Fiberhome/烽火 from manual `sessionid` entry to username/password login with live `base_info` dashboard data.

## Release Assets

| Asset | Use |
| --- | --- |
| `CPEManager-android-v0.3.1-release.apk` | Recommended Android install package for phone testing. |
| `CPEManager-android-v0.3.1-debug.apk` | Debug Android build for development troubleshooting. |
| `CPEManager-macos-arm64-v0.3.1-app.zip` | macOS Apple Silicon Tkinter desktop app bundle. |
| `CPEManager-web-v0.3.1.zip` | Static Flutter Web/PWA build output. |
| `cpemanager-0.3.1-py3-none-any.whl` | Python package wheel for CLI and desktop entrypoints. |
| `SHA256SUMS.txt` | SHA-256 checksums for all release assets. |

## Highlights

- Fixed Huawei login token handling for devices that return `challenge_login` error code `125003`.
- Huawei clients now prefer `/api/webserver/SesTokInfo`, use separate tokens for `challenge_login` and `authentication_login`, and keep `/api/webserver/token` as a fallback.
- Fiberhome now logs in with username/password using `get_refresh_sessionid` and `app_do_login`.
- Fiberhome dashboard now reads `app_get_base_info` and displays live signal, traffic, device, TAC/NCGI, MCS, MIMO, AMBR, temperature, software version, WorkMode, and neighbor rows.
- Fiberhome snapshot now reads `app_get_airplane`.
- Local capture folders are ignored so HAR files and temporary login scripts are not committed accidentally.

## Install Android

```bash
adb install -r CPEManager-android-v0.3.1-release.apk
```

The Android package is `com.cpemanager.app`, version `0.3.1`.

## Install Python CLI

```bash
python -m pip install cpemanager-0.3.1-py3-none-any.whl
cpemanager --help
cpemanager-desktop --version
```

## Remaining Limits

- Real Huawei and Fiberhome hardware write tests still need to be performed by a user on the CPE LAN.
- Android release signing is development-grade and not yet Play Store/AppGallery ready.
- iOS/macOS Flutter native builds still require full Xcode and CocoaPods on this Mac.
- Windows artifacts must be built on Windows.
- HarmonyOS/OpenHarmony needs a separate OpenHarmony-SIG Flutter SDK or ArkTS validation pass.
